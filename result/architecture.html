AdaptiveLossWeighting

losses/adaptive_loss.py — per-sample Σwᵢ(x)·lossᵢ
  1. StatisticalFeatureExtractor → 6 features (variance, skew, kurtosis, acf₁, spectral entropy, trend strength), EMA z-scored momentum .9, then .detach()
  2. WeightGeneratorMLP: 6 → 64 → 3, softmax T 1.5, floor-rescaled min .05
  3. MSE / MAE / Trend computed per sample, each EMA-normalized to scale ≈ 1 momentum .9
  4. total = Σ wᵢ·norm_lossᵢ − entropy_coef .3·H(w)
↓ one scalar per config → loss.backward() → stage 04

Where this stands: on ETTh1 (pred_len 96), the adaptive path converges to weights close to an equal split (w_mse ≈ .40 / w_mae ≈ .35 / w_trend ≈ .25) and roughly ties the plain equal-weight fixed baseline — MSE 9.165 vs. 9.149 — rather than clearly beating it. No strong per-sample heterogeneity has shown up yet on this series. The ETTh2 ablation (currently running) tests whether a more volatile dataset gives the per-sample weighting something real to react to.
