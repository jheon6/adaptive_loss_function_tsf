01

초기화 — 아무것도 모르는 상태에서 시작

weight_generator/mlp_generator.py:68-78

WeightGeneratorMLP._init_weights()가 마지막 Linear layer의 weight와 bias를 전부 0으로 초기화한다. 그러면 첫 forward에서 logits = Linear(64→3)(...) = 0이 되고, softmax(0 / 1.5) = [1/3, 1/3, 1/3].

학습 시작 시점엔 무조건 균등 가중치에서 출발한다 — "어떤 loss를 더 믿어야 할지 아직 모르니 일단 공평하게"라는 설계.
02

Forward 한 스텝, 실제 수치와 함께

features/statistical_extractor.py · losses/adaptive_loss.py · weight_generator/mlp_generator.py

배치 하나가 들어올 때 (trainer.py:108-114 → adaptive_loss.py:119-158) 일어나는 4단계.

  1. 피처 추출 + 정규화
    x_enc (B,336,C) → 6개 통계량(variance, skew, kurtosis, acf1, spectral_entropy, trend_strength) → 채널 평균 → raw features (B,6) 학습 중이면: running_mean/std를 EMA로 갱신 (momentum=0.9) → normalized = (features - running_mean) / (running_std + eps)

    이 EMA는 학습 가능한 파라미터가 아니라 buffer다 — gradient로 최적화되는 게 아니라 지금까지 본 배치들의 평균/표준편차를 누적 추적하는 통계치일 뿐이다.

  2. detach — 여기서 그래프가 끊긴다
    weights = self.weight_generator(features.detach())

    features는 x_enc로부터 나온 미분 가능한 텐서지만 .detach()로 그래프를 잘라버린다. feature extractor 쪽으로는 gradient가 절대 안 흐른다 — 학습되는 파라미터는 오직 weight_generator.net 안의 2개 Linear + LayerNorm뿐이다.

  3. MLP forward
    normalized features (B,6) → Linear(6→64) → LayerNorm(64) → ReLU → Dropout(.1) → Linear(64→3) = logits (B,3) → softmax(logits / 1.5) = raw_weights (B,3), 합=1 → floor rescale: w = raw_weights × (1 − 3×0.05) + 0.05 = raw_weights × 0.85 + 0.05

    마지막 floor 연산이 이번에 추가한 안전장치다 — softmax가 아무리 쏠려도 (예: raw_weights=[.999,.0005,.0005]) 최종 w는 [.899,.0504,.0504]로, 어떤 컴포넌트도 5% 밑으로는 내려가지 않는다.

  4. loss 계산 + 정규화 → 최종 스칼라
    raw_losses = [MSE, MAE, Trend](pred, target) (B,3) norm_losses = raw_losses / loss_running_mean (EMA, momentum=0.9) weighted = (w * norm_losses).sum(dim=1).mean() # 배치 평균 entropy = -(w * log(w)).sum(dim=1).mean() total = weighted - 0.3 * entropy
03

Backward — 두 힘의 줄다리기

total.backward()가 호출되면 체인룰을 타고 w(MLP 출력)까지 올라간다. floor rescale은 선형식이라 미분에 상수 0.85가 곱해질 뿐 방향은 안 바뀐다. 핵심은 softmax 이전 raw_w에 대한 gradient:

∂total/∂raw_w_i = norm_loss_i − 0.3 × (−log(w_i) − 1)

직관적으로 풀면 서로 반대 방향으로 당기는 두 힘의 합이다.

collapse pressure

norm_loss 항

이 성분의 (정규화된) loss가 클수록 w_i를 늘리면 total이 커지므로, gradient는 "이 컴포넌트의 weight를 줄여라"는 방향을 만든다. 반대로 지금 상대적으로 loss가 작은 컴포넌트는 "weight를 늘려도 손해가 적다"는 유인이 생긴다 — 그대로 두면 한쪽으로 몰빵(collapse)하는 힘이다.

restoring force

엔트로피 항

w_i가 0이나 1에 가까울수록 −log(w_i)가 극단값을 가지므로, 이 항의 gradient는 w를 균등 방향으로 되돌리는 힘을 만든다. entropy_coef = 0.3이 이 되돌리는 힘의 세기다.

최종 w는 이 두 힘의 줄다리기 결과다. 여기에 floor(0.05)가 세 번째 안전장치로 얹혀서, 두 힘의 균형이 한쪽으로 크게 기울어도 구조적으로 완전히 0이 되는 걸 막는다.

이 gradient는 Linear(64→3)의 weight/bias, 그리고 체인룰을 타고 LayerNorm, Linear(6→64)의 weight/bias까지 전부에 전달된다 — "어떤 피처 조합(변동성 높음 + 추세 강함 등)일 때 어떤 컴포넌트를 더/덜 신뢰할지"의 매핑 자체가 이 gradient로 조금씩 조정된다.

04

Optimizer — 백본과 다른 속도로

trainer/trainer.py

같은 total.backward() 한 번으로 백본과 MLP가 동시에 gradient를 받지만, 파라미터 그룹을 분리해서 서로 다른 학습률로 갱신한다.

backbone.parameters()lr = 1e-4
weight_generator.parameters()lr = 1e-4 × 0.5 = 5e-5
gradient clippingclip_grad_norm_(all, max_norm=1.0)
optimizerAdam, weight_decay 1e-5

MLP가 상대적으로 더 예민하고 불안정한 부분이라, 백본이 급하게 바뀌는 통계량을 MLP가 너무 빨리 뒤쫓다가 진동하는 걸 완화하려는 목적이다.

05

여러 epoch에 걸쳐 실제로 일어나는 일

초반 epoch — w는 균등([1/3,1/3,1/3] 근처)에서 시작, EMA(피처·loss)가 아직 실제 스케일을 잘 못 따라잡은 상태라 정규화가 다소 부정확 → weight가 다소 요동친다.

중반 — EMA가 안정화되며 정규화가 정확해지고, 두 힘(loss 압력 vs 엔트로피)의 균형점 근처로 w가 서서히 수렴한다.

ETTh1에서 실제로 확인된 값 (45 epoch 시점, early stopping 직전):

w_mse
≈ 0.40
w_mae
≈ 0.35
w_trend
≈ 0.25

균등(각 0.33)에서 크게 벗어나지 않은 채 정착한다.

즉 지금 설정에서 MLP가 배우는 최종 결과는 "샘플마다 크게 다른 정책"이라기보다 "대체로 균등에 가까운, 살짝 mse/mae 쪽으로 치우친 고정적인 정책"에 가깝다 — 이게 mse_mae_trend_fixed(고정 균등 ⅓/⅓/⅓)와 성능이 거의 같았던 (MSE 9.165 vs 9.149) 이유와 직접 연결된다.
